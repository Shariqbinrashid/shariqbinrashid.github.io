🎯 What is a Forward Deployed Engineer?

A Forward Deployed Engineer (FDE) is a unique hybrid role that sits at the intersection of cutting-edge AI engineering and hands-on customer delivery. Unlike traditional software engineers who build products in isolation, FDEs embed directly with strategic customers to architect, build, and deploy production AI systems that solve real business problems.

At companies like Anthropic, FDEs work within customer systems to build production applications with Claude models, deliver technical artifacts like MCP servers and agent skills, and provide white-glove deployment support in enterprise environments. You'll maintain deep knowledge of the latest LLM capabilities while building long-term relationships with customers and traveling 25-50% of the time.

This isn't just coding — it's combining engineering excellence with consulting skills, customer discovery, and the ability to thrive under ambiguity in complex organizations.

📅 8-Week Execution Plan

The fastest path from where you are to interview-ready. Follow this week by week.

Week 1

API Fundamentals

Rebuild an existing app on a frontier API. Add tool use + prompt caching. Ship something you can demo.

Week 2

MCP Server

Ship one MCP server publicly on GitHub. Pick an enterprise API (Jira-like, Slack-like). Both stdio + HTTP.

Week 3

Agent Skills

Read 3 reference skills from agentskills.io. Ship one custom skill that composes with your MCP server.

Week 4

Evaluation Harness

Add eval framework to Week 1 project. Get baseline metrics. Use Inspect AI or Braintrust.

Week 5

Multi-Agent System

Build 1 orchestrator + 2 sub-agents. Pick a workflow that needs multiple steps (research → analysis → output).

Week 6

Production Deploy

Deploy your agent to Bedrock or Vertex. Document security model, data flow, and observability setup.

Week 7

Vertical Depth

Pick one vertical (fintech, healthcare, legal). Build domain-shaped demo on top of existing infrastructure.

Week 8

Polish & Document

Write up 2-3 projects as case studies. Practice explaining them out loud. Prep for interviews.

⚠️ Don't Over-Invest In

Save your time and energy. These won't help you land the role.

❌ Building Vector DBs from Scratch

The job is about deploying LLMs well, not winning RAG benchmarks. Use Pinecone/Weaviate and focus on the retrieval strategy.

❌ Fine-tuning

Top frontier models aren't user-fine-tunable. Knowing why prompting + context engineering replace fine-tuning is more valuable than doing it.

❌ Framework Depth for Its Own Sake

Know LangChain/LlamaIndex/CrewAI exist. Don't build your demos on heavyweight frameworks — senior engineers prefer thin, direct API calls.

❌ Chasing Every Model Release

Frontier moves weekly. Build for capability classes (reasoning/tool-using/cheap-and-fast/multimodal), not specific model names.
