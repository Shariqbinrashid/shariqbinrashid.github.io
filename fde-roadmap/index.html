FDE Production Path

Real-world deployment roadmap for Forward Deployed Engineers. No theory. No tutorials. Just production systems, customer problems, and enterprise scale.

Phases
1

Production LLM Engineering

Ship a real app with actual users. Not a demo. Not a prototype. Something that breaks when you push bad code.

Week 1-2

Production Chat Agent with Tools

Build a Claude-powered agent that calls 3+ real APIs (not mocks). Add prompt caching to cut costs 80%. Deploy to Vercel/Railway with logs. Share the URL publicly. If it crashes, you fix it.

Claude API Tool Use Prompt Caching Production Deploy
Week 2-3

Cost Optimization Kill Sheet

Take a real LLM app spending $500/month. Get it to $50/month without degrading quality. Document: batch API usage, prompt compression, caching strategy, model selection, streaming optimization. Show the billing receipts.

Cost Analysis Batch API Optimization Production Metrics
2

Agent Infrastructure

Build systems that other engineers can use. MCP servers, agent skills, deployment patterns. This is where FDEs live.

Week 3-4

Enterprise MCP Server

Build an MCP server for a real enterprise API (Jira/Salesforce/ServiceNow clone). Support stdio + HTTP transports. Add auth (OAuth/API keys). Write docs a PM could follow. Get 3 people to use it. Handle their bug reports.

MCP Protocol OAuth TypeScript Documentation
Week 4-5

Multi-Agent Orchestrator

Build an orchestrator + 2 specialist sub-agents. One does research, one writes code, orchestrator routes and synthesizes. Add handoff logic. Deploy it. Show it solving a real problem (not "write me a poem").

Agent Architecture Sub-agents Routing Logic LangGraph
3

Enterprise Deployment

The part that separates hobbyists from FDEs. Security, compliance, audit logs, SSO, RBAC. The boring stuff that customers actually pay for.

Week 6-7

AWS Bedrock / GCP Vertex Deploy

Take your Week 1 agent. Deploy it on AWS Bedrock or GCP Vertex. Add: SSO (SAML/OIDC), RBAC (3 roles minimum), audit logging (who asked what when), cost allocation tags. Document the security model. Run it for a week.

AWS Bedrock SSO RBAC Audit Logs
Week 7-8

Production Incident Drill

Your production app just went down. Costs spiked 10x. A customer is waiting on Zoom. You have 30 minutes. Set up: Grafana dashboard, alert rules, runbook. Simulate: prompt injection, rate limit hit, model degradation. Fix it. Write the post-mortem.

Observability Grafana Incident Response Post-mortems
4

Evaluation & Quality

No evals = no production. Customers need numbers. "It works" doesn't cut it. Build eval harnesses that prove your agent isn't hallucinating.

Week 9-10

Eval Harness from Scratch

Pick one of your agents. Build a golden dataset (50+ examples). Implement: LLM-as-judge, pairwise comparison, rubric scoring. Set up CI/CD that fails if eval score drops >5%. Run it nightly. Show the regression you caught.

Inspect AI LLM-as-Judge Golden Datasets CI/CD
Week 10-11

Adversarial Eval Suite

Build 20+ adversarial test cases for your agent: prompt injection attempts, jailbreaks, edge cases, malformed inputs, hallucination triggers. Add guardrails. Measure: false positive rate, false negative rate, latency impact. Document what works.

Security Testing Guardrails Prompt Injection Red Teaming
5

Customer-Facing Delivery

The consulting half. Discovery, scoping, demos, stakeholder management. This is what FDE interviews test for.

Week 11-12

Enterprise Case Study

Pick a vertical (fintech/healthcare/legal). Design a full deployment: customer profile, problem statement, architecture diagram, RAG strategy, eval plan, security model, cost estimate, success metrics. Write it like a one-pager for a VP.

Discovery Architecture Business Case Stakeholder Mgmt
Week 12

Live Demo + Q&A

Record a 10-minute demo of your best project. Explain: the business problem, your solution, the tech stack, eval results, cost breakdown, what you'd do differently. Practice talking to executives, not engineers. Post it publicly. Get feedback.

Communication Demos Presentation Executive Presence

⚠️ Don't Waste Time On

  • ❌ Building vector DBs from scratch - Use Pinecone/Weaviate. FDEs deploy, not research.
  • ❌ Fine-tuning models - Frontier labs don't expose it. Context engineering > fine-tuning.
  • ❌ Framework rabbit holes - Know LangChain exists. Use thin abstractions. Direct API calls win.
  • ❌ Chasing model releases - Build for capability classes, not model names.
  • ❌ Tutorial hell - Stop watching videos. Ship something. Break it. Fix it. Repeat.
